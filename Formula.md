# Разбор скрипта: формула вероятности плей-офф и механика броска кубика

---

## 🎲 Часть 1 — Как подбрасывается кубик (симуляция одного матча)

Функция `simulate_remaining_games()`. Для каждого незыгранного матча:

### Шаг 1 — Составной рейтинг команды

\[
R_{team} = ELO_{norm} \cdot 0{,}50 + WinPct \cdot 0{,}25 + SoS \cdot 0{,}15 + SoV \cdot 0{,}10
\]

Где:
* **ELO\_norm** — нормализованный ELO: `(elo - 1000) / 400`, зажатый в `[0, 1]`
* **WinPct** — текущий процент побед команды
* **SoS** — сила прошлого расписания (из CSV)
* **SoV** — сила побеждённых соперников по рангу: \(1 - \frac{\bar{rank} - 1}{31}\)

### Шаг 2 — Базовая вероятность победы хозяев

\[
P_{home}^{base} = \frac{R_{home}}{R_{home} + R_{away}}
\]

### Шаг 3 — Поправки

| Поправка | Формула | Значение |
|---|---|---|
| Преимущество поля | \(P_{home} \mathrel{+}= 0{,}02\) | +2% |
| Серия побед хозяев (≥3) | \(P_{home} \mathrel{+}= 0{,}03\) | +3% |
| Серия побед гостей (≥3) | \(P_{home} \mathrel{-}= 0{,}03\) | −3% |
| Дивизионный матч | \(P_{home} = P_{home} + (0{,}5 - P_{home}) \cdot 0{,}15\) | регрессия 15% к 50/50 |

### Шаг 4 — Жёсткий кэп вероятности

\[
P_{home} = \text{clamp}(P_{home},\ 0{,}25,\ 0{,}75)
\]

Ни одна команда не может иметь больше 75% или меньше 25% шансов на победу.

### Шаг 5 — Сам бросок кубика

Генерируется одно случайное число `r = random.random()` ∈ [0, 1):

```
r ∈ [0 ; 0.003)                          → НИЧЬЯ (tie_score = randint(10,27))
r ∈ [0.003 ; 0.003 + P_home·0.997)       → ПОБЕДА ХОЗЯЕВ
r ∈ [0.003 + P_home·0.997 ; 1)           → ПОБЕДА ГОСТЕЙ
```

Формально три зоны на отрезке [0, 1):

\[
\underbrace{[0,\ T)}_{\text{ничья}} \quad \underbrace{[T,\ T + P_h \cdot (1-T))}_{\text{хозяева}} \quad \underbrace{[T + P_h \cdot (1-T),\ 1)}_{\text{гости}}
\]

где \(T = 0{,}003\) (константа `TIE_PROBABILITY`).

Счёт при победе:
* Победитель: `randint(17, 35)`
* Проигравший: `randint(7, score_winner - 1)`

## 📊 Часть 2 — Формула подсчёта шансов на плей-офф

Функция `calculate_playoff_probability_simulation()` — **метод Монте-Карло**:

\[
P_{playoff}(team) = \frac{\text{кол-во симуляций, где team попал в плей-офф}}{N_{sim}} \times 100\%
\]

Аналогично для дивизиона и бай-вика:

\[
P_{div}(team) = \frac{N_{div}}{N_{sim}} \times 100\%, \qquad P_{bye}(team) = \frac{N_{bye}}{N_{sim}} \times 100\%
\]

По умолчанию \(N_{sim} = 10\,000\).

### Что происходит внутри каждой симуляции:

```
Для каждой из N симуляций:
  1. simulate_remaining_games()  → бросить кубик по каждому матчу
  2. determine_playoff_teams()   → определить 7 команд плей-офф (4 дивизиона + 3 wild card)
  3. Если team в списке → playoff_count += 1
```

### Финальный кэп вероятности (`cap_probability`)

После расчёта применяется логика математической определённости:

| Статус | Итоговая вероятность |
|---|---|
| `clinched` (математически гарантировал) | **100%** |
| `eliminated` (математически выбыл) | **0%** |
| Симуляция дала 100% | Урезается до **99,9%** |
| Симуляция дала 0% | Урезается до **0,1%** |
| Всё остальное | Берётся как есть |

## 🏆 Как определяются 7 мест плей-офф (`determine_playoff_teams`)

```
Для каждой конференции (AFC / NFC):
  1. 4 победителя дивизионов (лучшие по win_pct + тайбрейкеры)
  2. Из них — 1 команда с бай-виком (1-й сид)
  3. Остальные команды конференции → сортировка по win_pct
  4. Топ-3 из них → wild card
  Итого: 4 + 3 = 7 команд
```

### Порядок тайбрейкеров при равном win_pct:

1. H2H (очные встречи)
2. Division % (для дивизионных тай)
3. Conference %
4. SoV (сила побеждённых)
5. SoS (сила расписания)
6. `random.choice()` — жеребьёвка
